## 1. 页面优先级

| 优先级 | 页面 | 路由 | 理由 |
|--------|------|------|------|
| P0 | 首页 | `/` | 门面，用户第一印象 |
| P0 | 产品列表页 | `/products` | 展示核心产品 |
| P1 | 产品详情页 | `/products/[slug]` | 深度了解产品 |
| P1 | 关于我们 | `/about` | 建立信任 |
| P1 | 联系我们 | `/contact` | 转化入口 |

### 开发顺序

```
Phase 1 (P0)
  ① 首页 + 通用组件（Header/Footer）
  ② 产品列表页

Phase 2 (P1)
  ③ 产品详情页
  ④ 关于我们页
  ⑤ 联系我们页

Phase 3 (收尾)
  ⑥ 内容填充
  ⑦ 测试 + 上线
```

## 4. 风险与依赖

### 4.1 风险评估

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 设计稿延迟 | 高 | 先用框架占位，内容后填 |
| 域名备案 | 中 | 初期用Vercel临时域名 |
| 内容文案不足 | 中 | 先用占位符，后续补充 |
| CI/CD配置问题 | 低 | 提前搭建CI，代码同步验证 |

### 4.2 依赖项

| 依赖 | 说明 | 责任人 |
|------|------|--------|
| 产品资料 | 3-5个产品的名称、描述、特性 | 运营/CEO提供 |
| 公司介绍文案 | 公司简介、团队成员信息 | CEO提供 |
| 域名（正式） | 正式域名（可延后） | CEO协调 |
| 设计稿 | UI设计（可先用框架） | 设计侧 |

### 4.3 不在本次范围

- 多语言支持
- 博客/新闻系统
- CMS内容管理
- 正式域名绑定
- 访问统计集成
